# 基于 Proteus & VOFA+ 的智能窗户/窗帘控制系统仿真指南

本指南旨在指导如何使用 **Proteus 8.x** 电路仿真软件与 **VOFA+** 串口上位机进行联合仿真调试，验证 STM32F103C8T6 固件代码的逻辑功能。

---

## 一、 仿真环境准备

在开始仿真前，请确保已安装并配置好以下软件：

1.  **Proteus 8.13 或以上版本**（需自带 STM32F103C8 仿真模型）。
2.  **Keil MDK-ARM v5.x**（用于编译生成 `.hex` 或 `.axf` 固件）。
3.  **VSPD (Virtual Serial Port Driver)**（虚拟串口软件，用于创建虚拟串口对）。
4.  **VOFA+ 上位机软件**（用于接收和显示波形数据）。

---

## 二、 Proteus 电路设计与接线指南

### 1. 关键器件搜索与选型表

在 Proteus 中新建工程，在 `Pick Devices` 窗口中搜索并添加以下器件：

| 原理图器件名称 | Proteus 检索关键字 | 说明 |
| :--- | :--- | :--- |
| **单片机核心** | `STM32F103C8` | 32位 ARM Cortex-M3 核心 |
| **步进电机驱动** | `ULN2003A` | 达林顿管阵列，用于放大电流驱动电机 |
| **步进电机** | `MOTOR-STEPPER` | 4线或6线单极性/双极性步进电机（模拟窗帘） |
| **舵机 (SG90)** | `MOTOR-SERVO` | 伺服电机（模拟窗户开闭） |
| **物理串口适配器**| `COMPIM` | 用于将仿真串口映射到电脑的物理/虚拟串口 |
| **雨滴传感器模拟**| `LOGICSTATE` 或 `SPDT` | 逻辑状态源或单刀双掷开关（模拟高低电平） |
| **独立按键** | `BUTTON` | 用于手动控制按键 |
| **上拉电阻** | `RESPACK-8` 或 `RES` | 用于按键及其他引脚的辅助拉高 |

### 2. 硬件引脚接线关系

按照下表在 Proteus 中进行引脚连线：

#### A. 串口联动（COMPIM）接线
*   `PA9 (USART1_TX)`  ➡️ 接 `COMPIM` 的 **RXD** 引脚
*   `PA10 (USART1_RX)` ➡️ 接 `COMPIM` 的 **TXD** 引脚
*   *注意：Proteus 中的 COMPIM 引脚功能定义为物理层，故应与单片机直连，无需交叉接线（即 TX-RXD，RX-TXD）。*

#### B. 步进电机（ULN2003A + 电机）接线
*   `PA4 (IN1)` ➡️ 接 `ULN2003A` 的 **IN1** (1引脚)
*   `PA5 (IN2)` ➡️ 接 `ULN2003A` 的 **IN2** (2引脚)
*   `PA6 (IN3)` ➡️ 接 `ULN2003A` 的 **IN3** (3引脚)
*   `PA7 (IN4)` ➡️ 接 `ULN2003A` 的 **IN4** (4引脚)
*   `ULN2003A` 的 **OUT1~OUT4** (16~13引脚) ➡️ 依次接 `MOTOR-STEPPER` 的 4 相输入引脚。
*   `ULN2003A` 的 **COM** 引脚 (9引脚) ➡️ 接 **+5V** 电源。
*   `MOTOR-STEPPER` 的公共端（若是6线电机）➡️ 接 **+5V** 电源。

#### C. 舵机（MOTOR-SERVO）接线
*   `PA1 (TIM2_CH2 PWM)` ➡️ 接 `MOTOR-SERVO` 的 **Control** (中间信号) 引脚。
*   舵机电源端 ➡️ 接 **+5V**，接地端 ➡️ 接 **GND**。

#### D. 雨滴传感器（模拟）接线
*   放置一个 `LOGICSTATE` 逻辑状态指示器。
*   `LOGICSTATE` 输出端 ➡️ 接 `PA0`。
    *   `0` (低电平) 代表：**有雨** (触发紧急关窗)。
    *   `1` (高电平) 代表：**无雨**。

#### E. 手动按键（BUTTON）接线
*   PB0, PB1, PB2, PB3 各自接一个 `BUTTON` 到 `GND`。
*   （由于代码中已使能 STM32 内部上拉 `GPIO_PULLUP`，仿真中可省略外部上拉电阻，直接单键接 GND）。

## 三、 Keil 编译与生成配置

1.  打开 Keil MDK 工程，点击 **Target Options (魔法棒图标)** ➡️ **Output** 选项卡。
2.  勾选 **Create HEX File**。
3.  点击 **C/C++** 选项卡，在 **Include Paths** 中确保添加了 `../User`。
4.  点击编译（Build），生成 `smartwindows.hex`。

## 四、 虚拟串口与 VOFA+ 联调步骤

要在电脑的 VOFA+ 上看到仿真里的波形，请按照以下步骤搭建虚拟串口通道：

1.  **创建虚拟串口对**：
    *   打开 **VSPD** 软件，点击 **Add pair**。它会成对生成两个相连的虚拟串口，例如 `COM1` 和 `COM2`。
2.  **配置 Proteus 中的 COMPIM**：
    *   在 Proteus 双击 `COMPIM` 器件。
    *   设置 **Physical Port** 为 `COM1`。
    *   设置 **Physical Baud Rate** 和 **Virtual Baud Rate** 均为 `115200`。
    *   数据位设为 `8`，校验位设为 `None`，停止位设为 `1`。
3.  **配置 VOFA+**：
    *   打开 VOFA+ 上位机，点击左上角的“➕”新建控件。
    *   在右侧连接设置中，选择串口号为 `COM2`，波特率 `115200`。
    *   选择数据协议为 **FireWater**。
    *   点击“打开串口”。
4.  **启动仿真**：
    *   双击 Proteus 中的 `STM32F103C8` 芯片，在 **Program File** 中加载 Keil 编译出的 `smartwindows.hex` 路径。
    *   在 **Crystal Frequency** 中输入 `8MHz`（注意：仿真时钟保持与配置相符）。
    *   点击 Proteus 左下角的 **Play (运行)** 按钮开始仿真。
    *   此时 VOFA+ 上将开始输出系统的 5 个通道曲线。

## 五、 避坑指南：BH1750 仿真卡死及规避方法

### ⚠️ 问题说明
由于 Proteus 中缺少通用的 `BH1750` 传感器 I2C 仿真组件，代码在执行 `HAL_I2C_Master_Receive()` 时会因为没有硬件芯片应答而陷入等待超时，这会导致整机逻辑卡死，主循环无法运行。

### 🛠️ 解决方案（代码免芯片仿真补丁）
在进行 Proteus 仿真时，我们可以通过代码上的 **条件编译** 绕过真实的 I2C 读写，直接提供一个随时间变化的模拟 Lux 光照值，或者通过读取一个 ADC 引脚电压来手动模拟光照。

目前已在 C 代码中提供了软件层面的规避准备。若仿真出现卡死，可将 [bh1750.c](file:///G:/AI_Projects/course_design/smartwindows/User/bh1750.c#L39-L54) 修改为以下仿真测试版：

```c
#define SIMULATION_MODE 1  /* 置1表示在Proteus中进行无芯片仿真测试 */

int BH1750_Read(float *lux_out)
{
#if SIMULATION_MODE
    /* 仿真模式: 产生一个 200lux 到 1000lux 循环变化的光照数据, 模拟昼夜交替 */
    static float fake_lux = 200.0f;
    static int direction = 1;
    
    if (direction) {
        fake_lux += 20.0f;
        if (fake_lux >= 1000.0f) direction = 0;
    } else {
        fake_lux -= 20.0f;
        if (fake_lux <= 100.0f) direction = 1;
    }
    *lux_out = fake_lux;
    HAL_Delay(10); // 模拟少许转换时间
    return 0; // 返回成功
#else
    /* 实体板模式: 进行真实的I2C读取 */
    uint8_t buf[2] = {0};
    if (HAL_I2C_Master_Receive(&hi2c1, BH1750_ADDR, buf, 2, 100) != HAL_OK)
    {
        return -1;
    }
    uint16_t raw = ((uint16_t)buf[0] << 8) | buf[1];
    *lux_out = (float)raw / 1.2f;
    return 0;
#endif
}
```

## 六、 仿真测试用例（下行控制命令）

在调试手动模式时，可以通过 VOFA+ 的发送区向单片机发送 Hex 指令：

| 功能 | 发送十六进制数据帧 (HEX) | 说明 |
| :--- | :--- | :--- |
| **切换到手动模式** | `AA 01 01 01 01` | 发送后，CH5（模式）波形变为 1.0 |
| **切换到自动模式** | `AA 01 01 00 00` | 发送后，系统恢复传感器驱动控制 |
| **手动开窗** | `AA 05 01 00 04` | 舵机将转动到 90° 挡位 |
| **手动关窗** | `AA 05 01 01 05` | 舵机将转回 0° 挡位 |
| **手动开帘（电机正转）**| `AA 04 01 00 05` | 步进电机运转开帘 |
| **手动关帘（电机反转）**| `AA 04 01 01 04` | 步进电机运转关帘 |
| **手动停止窗帘** | `AA 04 01 02 07` | 电机立即停止转动 |
| **查询系统状态** | `AA 06 00 C3` | 强制单片机立即上传当前通道数据 |
